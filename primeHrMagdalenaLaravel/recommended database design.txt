✅ RECOMMENDED DATABASE DESIGN
👤 1. employees (MAIN TABLE)

This is the center of everything

employees
- id (PK)
- employee_id
- first_name
- middle_name
- last_name
- suffix
- birth_date
- place_of_birth
- sex
- civil_status
- citizenship
- height
- weight
- blood_type
- mobile_number
- email
- created_at

👉 💡 Keep core identity here

🏠 2. addresses

👉 Because address is complex (don’t store in one field)

addresses
- id (PK)
- employee_id (FK)
- type (residential / permanent)
- house_no
- street
- barangay
- city
- province
- zip_code

government_ids
- id (PK)
- employee_id (FK)
- gsis_no
- philhealth_no

- tin_no
- license_no

👉 One employee can have MANY (elementary → college → grad)

educations
- id (PK)
- employee_id (FK)
- level
- school_name
- degree
- year_graduated
- honors

eligibilities

work_experiences

trainings
