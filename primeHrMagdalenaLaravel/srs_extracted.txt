B. Sample Use Case: UC-01 User Login
Use Case ID: UC-01Use Case Name: User LoginPrimary Actor: Registered User (e.g., Citizen, LGU Staff, Administrator)Stakeholders and Interests:
User – Wants secure and convenient access to the system.
System Owner / LGU – Wants only authorized users to access protected functions and data.
Brief Description:This use case describes how a registered user authenticates to the system using valid credentials to gain access to authorized features.
Preconditions:
The user account is registered and active.
The system is available, and the login page is accessible.
Postconditions:
Success: User is authenticated, an active session is created, and the user is redirected to the appropriate landing page (e.g., dashboard).
Failure: User remains unauthenticated; no secure resources are accessible; relevant error or validation messages are displayed.
Basic Flow (Main Success Scenario)
User navigates to the login page.
System displays the login form (username and password fields, plus optional “Remember Me”).
User enters a valid username and password.
User clicks the Login button.
System validates the credentials.
System creates a user session and records the login event in the audit log.
System redirects the user to the dashboard or rolebased home page.
Alternative / Exception Flows
A1 – Invalid Credentials
3a. User enters incorrect username and/or password.4a. User clicks Login.5a. System detects invalid credentials.6a. System displays an error message (e.g., “Invalid username or password”) without revealing which field is wrong.7a. System allows user to try again within configured limits.
A2 – Missing Required Fields
3b. User leaves username or password blank and clicks Login.4b. System detects missing data.5b. System displays validation messages indicating required fields and does not proceed with authentication.
A3 – Account Locked After Multiple Failed Attempts
1c–4c. User repeatedly submits invalid credentials for the same account.5c. System counts failed attempts and reaches the lockout threshold.6c. System locks the account and displays an appropriate message (e.g., “Your account has been locked. Please contact support or reset your password.”).7c. System logs the lockout event for audit and security review.
A4 – Remember Me (Optional)
3d. User ticks Remember Me before logging in with valid credentials.4d. System creates a persistent session token according to security policy.5d. On subsequent visits within the allowed period, system recognizes the user and logs them in automatically, subject to security constraints.
